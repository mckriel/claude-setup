# SENIOR JUDGEMENT FRAMEWORK
1. FRICTION - Will this increase or reduce friction for someone who did not write it?
2. ISOLATION - Can this code be tested, swapped or removed independantly.
3. CONVENTION - Does this fit into existing conventions or patterns in the codebase?

# PROMOTABLE ELEGANCE
1. Reduce Risk
2. Save money or time
3. Enable revenue
4. Prevent incidents that leadership cares about

# DESIGN QUESTIONS
1. What business problem does it solve?
2. What happens if we dont do this?
3. What is the smallest acceptale solution?
